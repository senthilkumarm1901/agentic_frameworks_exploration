## 1. Goal & Concept Mapping

Replicate the existing LangGraph Pattern 1 agent using the **Strands Agents SDK** while following Strands best practices.

| LangGraph Concept | Strands Equivalent | Strands Doc Reference |
|---|---|---|
| `create_react_agent()` | `Agent()` constructor -- the agent loop is built-in | [Agent Loop](https://strandsagents.com/docs/user-guide/concepts/agents/agent-loop/) |
| LangGraph MCP client (stdio) | `MCPClient` + `stdio_client` from `mcp.client.stdio` | [MCP Tools](https://strandsagents.com/docs/user-guide/concepts/tools/mcp-tools/) |

| Manual metrics capture | `AgentResult.metrics` + `get_summary()` | [Metrics](https://strandsagents.com/docs/user-guide/observability-evaluation/metrics/) |
| Callback / streaming | `callback_handler=None` (suppress stdout) | [Callback Handlers](https://strandsagents.com/docs/user-guide/concepts/streaming/callback-handlers/) |

## 6. File: `src/agent.py`

This is the core file. It creates the Strands agent with MCP tools via stdio transport, runs the query, and extracts structured metrics.

### Key Strands patterns used:
- **Managed MCP Integration**: `MCPClient` is passed directly to `Agent(tools=[mcp_client])`. The agent manages the MCP lifecycle. However, since we run single queries and exit, we use the **Manual Context Management** pattern with `with` statement to ensure clean shutdown.
- **OllamaModel**: Configured with `host` and `model_id` from environment variables.
- **callback_handler=None**: Suppresses all streaming output to stdout.
- **AgentResult.metrics**: Provides `get_summary()`, `cycle_durations`, `tool_metrics`, `accumulated_usage`.

```python
"""Strands agent with MCP tools -- core implementation."""

import json
import os
import time
from typing import Any

from strands import Agent
from strands.models.ollama import OllamaModel
from strands.tools.mcp.mcp_client import MCPClient
from mcp.client.stdio import stdio_client, StdioServerParameters

from .prompts import SYSTEM_PROMPT


def _get_mcp_server_command() -> list[str]:
    """Return the command to start the MCP server.

    The MCP server is expected to be a FastMCP server defined in the shared
    data directory. Adjust the path as needed for your project layout.

    Returns:
        A list of command-line arguments to start the MCP server.
    """
    # -- IMPORTANT --
    # Update this path to point to your actual MCP server script.
    # In the LangGraph version this was: python _shared/mcp_server.py
    script_dir = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
    server_path = os.path.join(script_dir, "..", "_shared", "mcp_server.py")
    return ["python", server_path]


def _build_ollama_model() -> OllamaModel:
    """Create an OllamaModel instance from environment variables.

    Environment variables:
        OLLAMA_MODEL      -- model ID  (default: "qwen3:8b")
        OLLAMA_BASE_URL   -- Ollama server URL (default: "http://localhost:11434")

    Returns:
        A configured OllamaModel instance.

    Reference:
        https://strandsagents.com/docs/user-guide/concepts/model-providers/ollama/
    """
    model_id = os.environ.get("OLLAMA_MODEL", "qwen3:8b")
    host = os.environ.get("OLLAMA_BASE_URL", "http://localhost:11434")

    return OllamaModel(
        host=host,
        model_id=model_id,
    )


def _build_mcp_client() -> MCPClient:
    """Create an MCPClient connected to the MCP server via stdio.

    The MCPClient wraps a factory function that returns a stdio connection.
    When used as a context manager (`with mcp_client:`), it starts and stops
    the MCP server subprocess automatically.

    Reference:
        https://strandsagents.com/docs/user-guide/concepts/tools/mcp-tools/

    Returns:
        An MCPClient instance (not yet connected -- call __enter__ or
        pass to Agent constructor).
    """
    server_cmd = _get_mcp_server_command()

    mcp_client = MCPClient(
        lambda: stdio_client(
            StdioServerParameters(
                command=server_cmd[0],
                args=server_cmd[1:],
            )
        )
    )
    return mcp_client


def _extract_tool_call_details(result_metrics: Any) -> tuple[int, list[str]]:
    """Extract tool call count and ordered tool names from metrics.

    The `tool_metrics` dict on EventLoopMetrics provides per-tool aggregates.
    Each value has an `execution_stats` with `call_count`.

    Args:
        result_metrics: The EventLoopMetrics object from AgentResult.

    Returns:
        A tuple of (total_tool_calls, list_of_tool_names_with_repeats).
    """
    summary = result_metrics.get_summary()
    tool_usage = summary.get("tool_usage", {})

    total_tool_calls = 0
    tools_used: list[str] = []

    for tool_name, stats in tool_usage.items():
        call_count = stats.get("call_count", 0)
        total_tool_calls += call_count
        # Add the tool name once per call for the tools_used list
        tools_used.extend([tool_name] * call_count)

    return total_tool_calls, tools_used


def run_agent(question: str) -> dict:
    """Run the Strands agent with MCP tools and return structured metrics.

    This function:
    1. Creates the OllamaModel from env vars
    2. Creates the MCPClient for the MCP server (stdio)
    3. Uses Manual Context Management (`with mcp_client:`) for lifecycle
    4. Discovers tools via `mcp_client.list_tools_sync()`
    5. Creates the Agent with tools, model, system_prompt, callback_handler=None
    6. Invokes the agent with the question
    7. Extracts answer + metrics from AgentResult
    8. Returns a JSON-serializable dict matching the LangGraph output format

    Args:
        question: The user question to answer.

    Returns:
        A dict with keys: answer, framework, pattern, llm_calls,
        total_duration_ms, tool_calls, tools_used.
    """
    model = _build_ollama_model()
    mcp_client = _build_mcp_client()

    wall_start = time.time()

    # -- Manual Context Management --
    # We use `with` so the MCP server subprocess is started and stopped
    # cleanly around our single query.
    # Ref: https://strandsagents.com/docs/user-guide/concepts/tools/mcp-tools/
    with mcp_client:
        # Discover tools from the MCP server
        tools = mcp_client.list_tools_sync()

        # Create the Strands Agent
        # - tools: list of MCP tool definitions
        # - model: OllamaModel instance
        # - system_prompt: defines agent behavior
        # - callback_handler=None: suppress streaming output to stdout
        agent = Agent(
            tools=tools,
            model=model,
            system_prompt=SYSTEM_PROMPT,
            callback_handler=None,
        )

        # Invoke the agent -- returns an AgentResult
        # Ref: https://strandsagents.com/docs/api/python/strands.agent.agent_result/
        result = agent(question)

    wall_end = time.time()
    total_duration_ms = round((wall_end - wall_start) * 1000)

    # -- Extract answer --
    answer = str(result)

    # -- Extract metrics --
    # result.metrics is an EventLoopMetrics instance
    # Ref: https://strandsagents.com/docs/user-guide/observability-evaluation/metrics/
    summary = result.metrics.get_summary()

    llm_calls = summary.get("total_cycles", 0)
    tool_calls_count, tools_used = _extract_tool_call_details(result.metrics)

    return {
        "answer": answer.strip(),
        "framework": "strands",
        "pattern": "agent_with_multiple_mcp_tools",
        "llm_calls": llm_calls,
        "total_duration_ms": total_duration_ms,
        "tool_calls": tool_calls_count,
        "tools_used": tools_used,
    }
```

### Design Decisions & Rationale

| Decision | Why |
|---|---|
| `with mcp_client:` (manual context) | We run one query then exit. Manual context ensures the MCP subprocess is cleaned up. |
| `mcp_client.list_tools_sync()` | Discovers tool schemas from the running MCP server and returns them as a list. |
| `Agent(tools=tools, ...)` | Pass discovered tool list directly. This is the **explicit tool list** pattern. |
| `callback_handler=None` | We capture output programmatically via `AgentResult`. No streaming to stdout. |
| `result.metrics.get_summary()` | Returns a dict with `total_cycles`, `tool_usage`, `accumulated_usage`, `traces`. |
| `str(result)` | `AgentResult.__str__()` extracts concatenated text from the final message. |
| Wall-clock timing | `time.time()` wraps the entire agent invocation for total duration. |

## 7. File: `src/main.py`

CLI entry point that matches the LangGraph interface exactly: `--task` and `--question` arguments, JSON output to stdout.

```python
"""CLI entry point for the Strands agent with MCP tools."""

import argparse
import json
import sys

from .agent import run_agent


def main():
    """Parse CLI arguments and run the agent."""
    parser = argparse.ArgumentParser(
        description="Strands Agent -- Pattern 1: Agent with Multiple MCP Tools"
    )
    parser.add_argument(
        "--task",
        type=str,
        default="country_analysis",
        help="Task identifier (for logging purposes)",
    )
    parser.add_argument(
        "--question",
        type=str,
        required=True,
        help="The question to ask the agent",
    )
    args = parser.parse_args()

    try:
        result = run_agent(args.question)
        # Output JSON to stdout -- this is what experiments.bash captures
        print(json.dumps(result, indent=4, ensure_ascii=False))
    except Exception as e:
        error_result = {
            "error": str(e),
            "framework": "strands",
            "pattern": "agent_with_multiple_mcp_tools",
        }
        print(json.dumps(error_result, indent=4, ensure_ascii=False), file=sys.stderr)
        sys.exit(1)


if __name__ == "__main__":
    main()
```

## 8. File: `experiments.bash`

Identical structure to the LangGraph version, with `FRAMEWORK="strands"`.

```bash
#!/usr/bin/env bash
set -euo pipefail

# =============================================================================
# CONFIGURATION
# =============================================================================
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

LOG_FILE="${SCRIPT_DIR}/logs.txt"
FRAMEWORK="strands"
PATTERN="agent_with_multiple_mcp_tools"

# Model/runtime settings for experiments.
# Override at runtime, for example:
#   OLLAMA_MODEL="qwen3:8b" ./experiments.bash
export OLLAMA_MODEL="${OLLAMA_MODEL:-qwen3.5:35b-a3b-coding-nvfp4}"
export OLLAMA_BASE_URL="${OLLAMA_BASE_URL:-http://localhost:11434}"

# =============================================================================
# STATIC METRICS (captured once per setup)
# =============================================================================
capture_static_metrics() {
  echo "================================================================================"
  echo "STATIC METRICS (captured once per setup)"
  echo "================================================================================"
  echo "timestamp: $(date -u +%Y-%m-%dT%H:%M:%SZ)"
  echo "framework: ${FRAMEWORK}"
  echo "pattern: ${PATTERN}"

  # Packaging size (KB -> MB)
  local venv_kb=$(du -sk .venv 2>/dev/null | cut -f1)
  local venv_mb=$(echo "scale=2; ${venv_kb:-0} / 1024" | bc)
  echo "packaging_size_mb: ${venv_mb}"

  # Lines of code (pattern's src/ only, excludes _shared)
  local loc=$(find src -name '*.py' -exec cat {} + 2>/dev/null | wc -l | tr -d ' ')
  echo "loc: ${loc}"

  # Dependency count from uv.lock
  local deps=$(grep -c 'name = ' uv.lock 2>/dev/null || echo "0")
  echo "dependency_count: ${deps}"
  echo ""
}

# =============================================================================
# RUN SECTION HEADER
# =============================================================================
ensure_run_section() {
  if ! grep -q "RUN METRICS" "${LOG_FILE}" 2>/dev/null; then
    echo "================================================================================" >> "${LOG_FILE}"
    echo "RUN METRICS (appended per execution)" >> "${LOG_FILE}"
    echo "================================================================================" >> "${LOG_FILE}"
  fi
}

# =============================================================================
# RUN EXPERIMENT
# =============================================================================
run_experiment() {
  local task="$1"
  local question="$2"

  echo "============================================================"
  echo "Task: $task"
  echo "Question: $question"
  echo "============================================================"

  # Append run header to logs
  {
    echo "--------------------------------------------------------------------------------"
    echo "run_timestamp: $(date -u +%Y-%m-%dT%H:%M:%SZ)"
    echo "task: ${task}"
    echo "question: ${question}"
    echo "model: ${OLLAMA_MODEL}"
  } >> "${LOG_FILE}"

  # Run and capture output
  local output
  local start_time=$(date +%s.%N)

  if output=$(uv run python -m src.main --task "$task" --question "$question" 2>&1); then
    local end_time=$(date +%s.%N)
    local duration=$(echo "$end_time - $start_time" | bc)

    echo "$output"

    # Append output to logs in readable format
    {
      echo "duration_seconds: ${duration}"
      echo "status: success"
      echo "output:"
      echo "$output" | sed 's/^/  /'
      echo ""
    } >> "${LOG_FILE}"
  else
    local end_time=$(date +%s.%N)
    local duration=$(echo "$end_time - $start_time" | bc)

    echo "ERROR: $output"

    {
      echo "duration_seconds: ${duration}"
      echo "status: error"
      echo "output:"
      echo "$output" | sed 's/^/  /'
      echo ""
    } >> "${LOG_FILE}"
  fi

  echo
}

# =============================================================================
# MAIN LOGIC
# =============================================================================
main() {
  echo "Running experiments with:"
  echo "  FRAMEWORK=$FRAMEWORK"
  echo "  PATTERN=$PATTERN"
  echo "  OLLAMA_MODEL=$OLLAMA_MODEL"
  echo "  OLLAMA_BASE_URL=$OLLAMA_BASE_URL"
  echo "  LOG_FILE=$LOG_FILE"
  echo

  # If logs.txt doesn't exist or is empty, capture static metrics first
  if [[ ! -s "${LOG_FILE}" ]]; then
    echo "Capturing static metrics (first run)..."
    capture_static_metrics > "${LOG_FILE}"
  fi

  # Ensure run section header exists
  ensure_run_section

  # Run experiments
  run_experiment "country_analysis" "What is the population density of Japan in people per square kilometer?"
  run_experiment "country_analysis" "What is the ratio of GDP per capita between the United States and Japan? Calculate it precisely"
  run_experiment "country_analysis" "What is the GDP per capita of Germany in trillion USD per million people?"
  run_experiment "country_analysis" "Which country has a larger population, India or China?"

  echo "All experiments finished."
  echo "Check logs in: ${LOG_FILE}"
}

# Run main
main "$@"
```

## 12. Troubleshooting Guide

| Issue | Fix |
|---|---|
| `ModuleNotFoundError: No module named 'strands'` | Run `uv sync` in the project directory |
| `ModuleNotFoundError: No module named 'strands.models.ollama'` | Ensure `strands-agents[ollama]` (not just `strands-agents`) is in deps |
| MCP server won't start | Check the `_get_mcp_server_command()` path -- it must point to your FastMCP server script |
| `ConnectionRefusedError` for Ollama | Ensure `ollama serve` is running and `OLLAMA_BASE_URL` is correct |
| Agent returns empty answer | Check that `callback_handler=None` is set (otherwise output goes to stdout, not `AgentResult`) |
| `tool_metrics` is empty | The model may not have called any tools. Check the system prompt and question. |
| Import error for `mcp.client.stdio` | Ensure `mcp` is in your `pyproject.toml` dependencies |

## 13. Differences from LangGraph Version

| Aspect | LangGraph | Strands |
|---|---|---|
| Agent creation | `create_react_agent(model, tools)` | `Agent(tools=tools, model=model, system_prompt=..., callback_handler=None)` |
| Model setup | `ChatOllama(model=..., base_url=...)` | `OllamaModel(host=..., model_id=...)` |
| MCP connection | `MultiServerMCPClient` with `__aenter__`/`__aexit__` | `MCPClient(lambda: stdio_client(...))` with `with` block |
| Async/sync | Fully async (`await`, `async with`) | Fully sync (`with`, direct calls) |
| System prompt | Separate message in conversation | `system_prompt` parameter on `Agent()` |
| Streaming control | Manual stream handling | `callback_handler=None` to suppress |
| Metrics | Manual counters in callbacks | `result.metrics.get_summary()` built-in |
| Dependencies | `langgraph`, `langchain-*`, `langchain-mcp-adapters` | `strands-agents[ollama]`, `mcp` |

## 14. Notes on the MCP Server

The MCP server itself (FastMCP with `country_lookup_tool` and `calculator_tool`) is **shared** between LangGraph and Strands implementations. It lives at `_shared/mcp_server.py` and does NOT need to be modified.

If your MCP server path differs, update the `_get_mcp_server_command()` function in `src/agent.py`.
